# 抽出ルール: CLAUDE.mdから何をどこに移すか

## 判断フローチャート

各セクション/段落に対して、上から順に判定する:

1. **削除対象か？** → コードから自明、git履歴で分かる、汎用ベストプラクティス → **削除**
2. **毎回必須か？** → ビルド/テストコマンド、致命的な禁止事項、プロジェクト概要 → **CLAUDE.mdに残す（1〜3行）**
3. **再利用ワークフローか？** → 手順化された開発フロー、CI/CD手順 → **skills/へ**
4. **プロジェクト固有の詳細か？** → 規約、アーキテクチャ、テスト戦略 → **docs/へ**

## 抽出先マッピング

### docs/ に移すもの

| 対象 | 移動先 | CLAUDE.mdに残す要約例 |
|------|--------|----------------------|
| コードスタイル規約（命名規則、フォーマット、lint設定の詳細） | `docs/code-style.md` | `- コードスタイル → docs/code-style.md` |
| アーキテクチャ解説（レイヤー構造、依存関係、設計判断） | `docs/architecture.md` | `- アーキテクチャ → docs/architecture.md` |
| テスト戦略（テスト分類、カバレッジ要件、モック方針） | `docs/testing.md` | `- テスト戦略 → docs/testing.md` |
| 環境構築手順（前提ツール、初期設定、認証設定） | `docs/setup.md` | `- 環境構築 → docs/setup.md` |
| 設計判断の記録（ADR的な内容） | `docs/decisions.md` | `- 設計判断 → docs/decisions.md` |
| API仕様・外部連携の詳細 | `docs/api.md` | `- API仕様 → docs/api.md` |

### skills/ に移すもの

| 対象 | 移動先 | CLAUDE.mdに残す要約例 |
|------|--------|----------------------|
| デプロイ手順 | `.claude/skills/deploy/SKILL.md` | `- デプロイ → /deploy スキル` |
| リリースフロー | `.claude/skills/release/SKILL.md` | `- リリース → /release スキル` |
| DBマイグレーション手順 | `.claude/skills/migrate/SKILL.md` | `- マイグレーション → /migrate スキル` |
| コードレビュー基準 | `.claude/skills/review/SKILL.md` | `- レビュー基準 → /review スキル` |

### 削除するもの

| 対象 | 理由 |
|------|------|
| 「変数名はcamelCaseを使う」等の、linterで検出できるルール | 自動化で担保される |
| 「コミットメッセージはConventional Commitsに従う」等の、git hooksで強制されるルール | 自動化で担保される |
| ファイル一覧やディレクトリツリーの詳細 | `ls`やglobで確認できる |
| 最近の変更履歴 | `git log`で確認できる |
| 一般的なベストプラクティス（DRY、SOLID等） | Claudeが既知 |
| 使用ライブラリの基本的な使い方 | Context7やドキュメントで確認できる |

## サイズ目安

| ファイル | 推奨行数 | 超過時のアクション |
|---------|---------|-------------------|
| CLAUDE.md | 30〜60行 | セクションを外部化 |
| docs/各ファイル | 50〜150行 | さらに分割を検討 |
| skills/SKILL.md | 50〜200行 | 参照ファイルに分離 |

## モノレポの場合

```
root/
  CLAUDE.md              ← プロジェクト全体の索引（共通ルール最小限）
  docs/                  ← 共通の詳細ドキュメント
  packages/
    api/
      CLAUDE.md          ← apiパッケージ固有の索引
      docs/              ← apiパッケージ固有の詳細
    web/
      CLAUDE.md          ← webパッケージ固有の索引
```

Claudeは親ディレクトリのCLAUDE.mdを自動で読み込むため、共通ルールはルートに、パッケージ固有はサブディレクトリに置く。重複しない。

## 既存構造への適応

プロジェクトに既に以下がある場合は、それに合わせる（新規作成しない）:

- `.claude/rules/` → ルール系はここに移す
- `.claude/skills/` → ワークフロー系はここに移す  
- `docs/` → ドキュメント系はここに移す
- `.cursor/rules/` → Cursor併用プロジェクトの場合は共存を意識
