伊東歯科口腔病院の歯式AI稼働状況レポートについて、次回定例向けの空シートを作成する。

## 対象ファイル
- 定例予定: `C:\Users\y-miyata\Box\電子カルテWG\電子カルテ協議事項管理表.xlsx`
- 報告ブック: `C:\Users\y-miyata\Box\歯式AI関連\伊東歯科口腔病院_歯式AI導入・更新手順と報告.xlsx`

## 引数
- 引数なし: 作業時点の Asia/Tokyo の日付を基準にする。
- `$ARGUMENTS` に `YYYY-MM-DD` が渡された場合は、その日付を「シート作成日」として扱う。

## 必須方針
- 会話・報告は日本語。
- 「コピーした」と報告できるのは、Excel 本体のシートコピー機能で複製した場合だけ。
- `openpyxl.copy_worksheet` など、ライブラリ独自のシート複製で代替しない。Excel COM が使えない場合は作業を止め、理由を報告する。
- 値を埋めずに作る。日別実績値と概要文は空欄にする。
- 変更前に報告ブックと同じフォルダへバックアップを作る。
- 完了報告では、実行した検証結果を先に示してから「完了」と述べる。

## 手順

1. 基準日を確定する。
   - 引数があれば `YYYY-MM-DD` として解釈する。
   - 引数がなければ JST の現在日を使う。
   - JST 取得は PowerShell の `[System.TimeZoneInfo]::ConvertTimeBySystemTimeZoneId([DateTime]::UtcNow, "Tokyo Standard Time")` を使い、推測しない。

2. 定例予定ブックから次回定例日を確認する。
   - `定例会予定` シートを読む。
   - Excel シリアル値・日付型・文字列日付のいずれでも解釈できるようにする。
   - 基準日以降で最も近い日付を「次回定例日」とする。
   - 確認した次回定例日を作業ログに出す。

3. 報告ブックからコピー元シートを特定する。
   - シート名が `稼働状況レポート(` で始まるものを対象にする。
   - 日付が最も新しいシートをコピー元にする。
   - コピー元シートの `K2` から前回集計範囲を読み取る。
   - 新しい集計開始日は、前回集計範囲の最終日の翌日。
   - 新しい集計終了日は、基準日の前日。
   - 新しいシート名は `稼働状況レポート(YYYY.M.D)`。日付は次回定例日。
   - `Q2` は `稼働状況レポート(YYYY/M/D)`。日付は次回定例日。
   - `K2` は `YYYY/M/D～YYYY/M/D`。日付は新しい集計範囲。

4. Excel 本体でシートをコピーする。
   - Windows の Excel COM を起動する。
   - 報告ブックを開く。
   - コピー元シートに対して Excel の `Worksheet.Copy` を使い、ブック末尾に複製する。
   - 複製後のシート名を新しいシート名に変更する。
   - 同名シートが既にある場合は上書きせず、状況を確認してから進める。

5. 新シートを空の報告テンプレートに調整する。
   - 日付行は、B列に集計開始日から集計終了日までを連続で入れる。
   - D列は各行 `=IF(B行="","",TEXT(B行,"aaa"))` にする。
   - 日別実績値の先頭セルを空欄にする。対象列は `E`, `H`, `K`, `N`, `R`, `T`。
   - `C5` の概要文を空欄にする。
   - `レポート期間計` 行の合計式を新しい日付行範囲に合わせる。
     - `E`: `=SUM(E開始:G終了)`
     - `H`: `=SUM(H開始:J終了)`
     - `K`: `=SUM(K開始:M終了)`
     - `N`: `=SUM(N開始:P終了)`
     - `R`: `=SUM(R開始:S終了)`
     - `T`: `=SUM(T開始:U終了)`
   - 使用率セルは `=IF(K合計行=0,"",N合計行/K合計行)` にする。
   - 印刷範囲を新しい最終行に合わせる。
   - 日数がコピー元と異なる場合の行削除・行追加は Excel 本体の行操作で行う。

6. 保存後に必ず読み戻し検証する。
   - 作成シートが存在すること。
   - 作成シートがブック末尾にあること。
   - `K2` が期待する集計範囲であること。
   - `Q2` が期待する表題であること。
   - 日付行の先頭・末尾が期待どおりであること。
   - 日別実績値対象セルに値が残っていないこと。
   - 合計式が新しい行範囲を参照していること。

7. レイアウト検証を必ず行う。
   - コピー元シートと新シートの列幅差分を確認する。
   - コピー元シートから日数差分の行操作を反映した期待値と、新シートの行高を比較する。
   - コピー元シートから日数差分の行操作を反映した期待値と、新シートの結合セル範囲を比較する。
   - 列幅差分、行高差分、結合セル差分が 0 であることを確認する。
   - 差分があれば完了扱いにせず、修正する。

- 集計範囲
- コピー元シート
- 実行した検証結果

  - ブック末尾確認
  - 日付範囲確認
  - 実績値空欄確認
  - 列幅差分
  - 行高差分
  - 結合セル差分
- バックアップファイル
- 残リスク
